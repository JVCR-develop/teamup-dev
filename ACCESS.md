# Управление доступом в проекте TeamUp Dev

## Настройка GitHub

### 1. Команды в организации

| Команда | Участники | Доступ к репо | Область ответственности |
|---------|-----------|---------------|------------------------|
| @teamup-dev/pm | Ты (PM) | Admin | Всё |
| @teamup-dev/devops | DevOps | Admin | Инфраструктура, CI/CD |
| @teamup-dev/backend | Бэкендер | Write | `/backend/*` |
| @teamup-dev/frontend | Дизайнеры-фронты | Write | `/frontend/*` |
| @teamup-dev/designers | Дизайнеры-фронты | Triage | Дизайн-токены, UI/UX |
| @teamup-dev/analytics | Аналитик | Read | Документация, метрики |
| @teamup-dev/everyone | Все | Read | Общие уведомления |

### 2. Правила работы с кодом

#### Ветки
- **main** — защищена, только через PR
- **develop** — основная ветка разработки
- **feature/*** — для новых фич
- **bugfix/*** — для исправлений
- **hotfix/*** — для срочных правок

#### Процесс
1. Создай ветку от `develop`
2. Внеси изменения
3. Создай Pull Request в `develop`
4. CODEOWNERS автоматически назначит ревьюверов
5. Получи апрув
6. Смержи

### 3. Кто что может менять

| Твоя роль | Можешь изменять | Не трогай без согласования |
|-----------|-----------------|---------------------------|
| **Бэкендер** | `/backend/`, `/docs/api/` | Инфраструктура, фронтенд |
| **Фронтендер** | `/frontend/` (кроме стилей дизайн-системы) | Бэкенд, API-эндпоинты |
| **Дизайнер** | `/frontend/src/assets/styles/`, `/frontend/public/images/`, `/docs/design/` | Логику компонентов, бэкенд |
| **Аналитик** | `/docs/analytics/`, `/docs/project/` | Код (без PR) |
| **DevOps** | Всё | - |
| **PM** | Всё | - |

### 4. Как получить доступ

1. Тебя должны добавить в GitHub организацию `teamup-dev`
2. Попроси PM добавить тебя в соответствующую команду
3. Настрой двухфакторную аутентификацию (2FA)
4. Добавь свой SSH ключ в настройках профиля

### 5. Частые вопросы

**Q: Могу ли я пушить прямо в main?**
A: Нет, ветка защищена. Только через Pull Request.

**Q: Сколько нужно ревьюверов?**
A: Минимум 1, но CODEOWNERS может запросить больше.

**Q: Что если мне нужно срочно исправить баг?**
A: Создай hotfix ветку, внеси изменения и PR с пометкой URGENT.

**Q: Как узнать, кто отвечает за конкретную папку?**
A: Посмотри в `.github/CODEOWNERS` или просто спроси в чате.
