# Что мы НЕ храним в Git (и почему)

## 1. Секреты и пароли
`.env` — здесь лежат пароли от БД, API ключи. Если они попадут в Git — злоумышленники взломают сервер.

## 2. Зависимости
`node_modules/` — 100+ МБ мусора, который можно восстановить командой `npm install`
`venv/` — то же самое, `pip install -r requirements.txt` решит проблему

## 3. Сгенерированные файлы
`*.pyc` — компилированные Python-файлы (меняются при каждом запуске)
`*.log` — логи (их место на сервере, не в репозитории)
`dist/` и `build/` — собранный фронтенд (генерируется при деплое)

## 4. Базы данных
`*.db`, `*.sqlite3` — БД не место в Git (только дампы для миграций)

## 5. Большие файлы
`*.pkl`, `*.h5` — ML модели могут быть гигабайтами. Git с ними не справится.
`data/` — сырые данные для обучения (храним отдельно в S3/Google Drive)

## 6. Личные настройки IDE
`.vscode/`, `.idea/` — у каждого свои настройки, не надо их навязывать другим
